# 用户网络自测指南

当登录或加载失败、且提示「网络错误」或「邮箱或密码错误」但您确认账号密码无误时，可能是当前网络、运营商、防火墙或 DNS 解析导致无法访问服务。请按下面步骤在**您自己的设备上**做一次自测，以便确认问题来源。

---

## 一、在 iOS 设备上可做的测试

### 1. 用 Safari 测试能否打开网站

- 在**同一网络下**（和打开 UHUB App 时相同的 Wi‑Fi 或移动数据），打开 **Safari**。
- 在地址栏输入并访问：
  - **https://www.uhub.help**
- **结果判断**：
  - 能正常打开页面 → 说明当前网络到我们服务器是通的，问题更可能是 App 配置或缓存，可尝试重装/更新 App 或清除 App 数据后再试。
  - 打不开、一直转圈或报错 → 多半是当前网络/运营商/防火墙或 DNS 问题，请继续下面步骤。

### 2. 换网络再试

- **关掉 Wi‑Fi，只用移动数据**再试一次（或反过来：关移动数据，只连 Wi‑Fi）。
- 或换一个 Wi‑Fi（例如换到手机热点、朋友家的 Wi‑Fi）。
- 若**换网络后能正常打开**，说明是原先那个网络（如公司/学校 Wi‑Fi、某运营商）对访问有限制或 DNS 异常。

### 3. 关闭 VPN / 代理

- 若您开启了 **VPN** 或 **代理**，请先关闭后再用 Safari 打开 https://www.uhub.help 或再试登录。
- 部分 VPN/代理会导致无法访问或解析我们的域名。

### 4. 检查 / 更换 DNS（iOS）

- 打开 **设置 → Wi‑Fi**，点击当前已连接 Wi‑Fi 右侧的 **(i)**。
- 下滑找到 **配置 DNS**，若为「手动」且列表里有不熟悉的地址，可改为 **自动**；或改为手动并添加：
  - `8.8.8.8`（Google）
  - `1.1.1.1`（Cloudflare）
- 保存后，再试 Safari 打开 https://www.uhub.help 或重新打开 App 登录。

### 5. 用 iOS 上的网络工具 App 测解析与连通性（可选）

- 在 App Store 搜索 **「Ping」「Network Analyzer」「DNS 查询」** 等工具。
- 在工具里：
  - **Ping** 或 **解析** 域名：`www.uhub.help`
- **结果判断**：
  - 能解析到 IP 且 ping 通 → 当前网络到我们服务器连通正常，若 App 仍失败，更可能是 App 端问题。
  - 无法解析或 ping 不通 → 当前网络/DNS/防火墙有问题，可结合「换网络、关 VPN、改 DNS」再测。

---

## 二、如有 Mac，可在终端里测（可选）

在终端执行（同一网络下）：

```bash
# 测试域名解析
nslookup www.uhub.help

# 测试能否连通（若 ping 被禁则可能无响应，属正常）
ping -c 3 www.uhub.help
```

- 若 `nslookup` 能返回 IP、`ping` 有回复，说明本机网络与 DNS 到我们服务器是通的。
- 若解析失败或 ping 不通，说明当前网络/运营商/DNS 或防火墙有限制。

---

## 三、反馈给技术支持时建议提供的信息

若自测后仍无法确定原因，可把下面信息提供给客服/技术支持，便于进一步排查：

- **设备**：例如 iPhone 14、iOS 17.x
- **网络**：当前是 Wi‑Fi 还是移动数据；若 Wi‑Fi，是家庭/公司/学校
- **运营商**：若为移动数据，所属运营商（如中国移动/联通/电信、香港/海外运营商）
- **自测结果**：Safari 能否打开 https://www.uhub.help；换网络/关 VPN/改 DNS 后是否有变化；若用过 Ping/Network Analyzer，是否能看到解析或 ping 通

---

以上步骤均为在**用户本地设备上**自行操作，用于确认是网络/运营商/防火墙或 DNS 解析异常，而不是服务端故障。
